### 🏥 Smart Health Care Insights
> *Python · Machine Learning · NLP · Sentiment Analysis · Flask · TF-IDF*

| What | Details |
|:-----|:--------|
| 🎯 **Problem** | Hospitals lacked scalable mechanisms to analyze patient feedback — reactive, not proactive quality improvement |
| 🔬 **Solution** | ML-powered feedback analysis system applying NLP & sentiment classification (positive / negative / neutral) in real time |
| ⚙️ **Pipeline** | Tokenization → Stopword Removal → TF-IDF Vectorization → Classification Model → Admin Dashboard |
| 📈 **Impact** | Automated feedback classification; enabled data-driven operational decisions in healthcare environments |

<br/>

### ✈️ Travel Planner with AI Chatbot
> *Python · AI/NLP · Flask · REST APIs · JSON · Intent Recognition*

| What | Details |
|:-----|:--------|
| 🎯 **Problem** | Travelers face information overload — needed one unified AI assistant for personalized trip planning |
| 🤖 **Solution** | Conversational chatbot leveraging intent recognition & entity extraction for real-time destination, hotel, and emergency recommendations |
| 🏗️ **Architecture** | Modular Flask REST backend with pluggable third-party travel API connectors and future ML expansion hooks |
| 📈 **Impact** | Dramatically reduced trip research time by consolidating planning, bookings, and emergency info in one interface |

<br/>

### 🚂 Railway Reservation System
> *Java · MySQL · JDBC · OOP · Swing GUI · DAO Pattern · MVC Architecture*

| 🖥️ **Frontend** | Swing-based GUI with form validation, error handling, and intuitive UX for both passengers and admin operators |
| 🏗️ **Design** | DAO Pattern + MVC-inspired architecture ensuring clean separation of concerns and long-term maintainability |
